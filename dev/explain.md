## Explain: $ARGUMENTS

### Structure

1. **Overview (1-2 sentences)**
   - What is this code's purpose?
   - What problem does it solve?

2. **Key Concepts**
   - What patterns/techniques are used?
   - What domain knowledge is needed?

3. **Step-by-Step Walkthrough**
   - Follow the main execution path
   - Explain each significant section
   - Use analogies where helpful

4. **Inputs & Outputs**
   - What does it receive?
   - What does it return/produce?
   - What side effects does it have?

5. **Dependencies**
   - What does it rely on?
   - What relies on it?

6. **Edge Cases & Error Handling**
   - How does it handle failures?
   - What assumptions does it make?

### Guidelines

- Explain the WHY, not just the WHAT
- Use plain language, avoid jargon
- If jargon is necessary, define it
- Include relevant code snippets with annotations
- Suggest improvements only if asked
